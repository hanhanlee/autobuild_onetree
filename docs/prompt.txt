Autobuild Onetree - Current Engineering Brief

This file replaces the original MVP prompt. The old text described an early design that no longer matches the deployed system.

Current deployment facts
- App root: /opt/autobuild
- Workspace root: /work/autobuild_workspace
- Jobs root: /work/autobuild_workspace/jobs
- Database: /work/autobuild_workspace/data/jobs.db
- Token root: /work/autobuild_workspace/secrets/gitlab
- Service user: autobuild
- Service group / login gate default: scm-bmc

Current runtime behavior
- FastAPI + Jinja2 app served by Uvicorn.
- Login uses PAM and only allows Linux users inside AUTOBUILD_ALLOWED_GROUP.
- Session storage uses Starlette SessionMiddleware with signed cookie.
- app.main loads .env, but systemd already sets AUTOBUILD_WORKSPACE_ROOT, AUTOBUILD_JOBS_ROOT, and AUTOBUILD_DB.
- Job runner is started as the autobuild service user via /opt/autobuild/runner/run_job.sh <job_id>.
- Runner is not launched with sudo -u <owner> in the current code path.
- User GitLab tokens are stored under /work/autobuild_workspace/secrets/gitlab/<username>.token.

Current documentation split
- DEPLOY_GUIDE.md: full install, redeploy, upgrade, systemd, nginx, .env, verification.
- USER_GUIDE.md: day-to-day UI operations, jobs, tokens, cleanup, common operator issues.

When updating docs or prompts, keep these points aligned
- Do not reintroduce sudoers / NOPASSWD instructions unless the runtime is changed back to that model.
- Do not describe workspace or DB defaults as /srv/autobuild/...; current shipped paths use /work/autobuild_workspace/...
- Do not claim .env alone can override AUTOBUILD_WORKSPACE_ROOT, AUTOBUILD_JOBS_ROOT, or AUTOBUILD_DB while the current systemd unit exports them.
- If deployment flow changes, update DEPLOY_GUIDE.md, USER_GUIDE.md, README.md, and this file together.

Known historical mismatches now fixed in docs
- Old prompt assumed ~/.autobuild/gitlab_token; current code uses /work/autobuild_workspace/secrets/gitlab/<username>.token.
- Old prompt assumed runner command arguments repo/ref/machine/target; current runner entrypoint only requires <job_id>.
- Old prompt assumed /srv/autobuild paths; current deployment uses /opt/autobuild and /work/autobuild_workspace.
